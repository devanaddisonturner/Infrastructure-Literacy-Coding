# Data Files

This directory contains the coded learning outcome data for the Infrastructure
Literacy curriculum content analysis.

## `layer2_outcomes.json`

JSON array of **431** individually coded learning outcomes across four construction
credentials in three national systems. Each object contains:

| Field | Type | Description |
|-------|------|-------------|
| `num` | int | Sequential row number (1–431) |
| `credential` | string | Source credential name |
| `unit` | string | Module or unit within credential |
| `outcome_id` | string | Official outcome identifier |
| `description` | string | Full text of the learning outcome |
| `crit_a` | int (0/1) | Distributional consequences criterion |
| `crit_b` | int (0/1) | Community health equity criterion |
| `crit_c` | int (0/1) | Environmental justice concepts criterion |
| `result` | string | "Present" or "Absent" |
| `notes` | string | Coder notes for borderline cases |

**Credential counts:**

| Credential | System | Coded level | N |
|---|---|---|---|
| NCCER Core Curriculum (6th ed.) | USA | Sub-objective | 103 |
| CA CTE Building & Construction Standards | USA | Performance indicator | 170 |
| CPC30220 Certificate III in Carpentry | Australia | Element of competency | 114 |
| City & Guilds 6706-23 Level 2 Diploma | UK | Learning outcome | 44 |
| **Total** | | | **431** |

**Finding:** Zero outcomes (0%) meet any of the three binary equity criteria.

## `borderline_cases.json`

JSON array of **8** objects documenting outcomes that required explicit coding
justification — for example, environmental regulations mentioned in compliance-only
contexts. Each entry explains why the outcome was coded Absent despite surface-level
keyword presence.
